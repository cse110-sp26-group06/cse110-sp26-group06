## Key Concepts Explained:

### Sprints:
- Break project timeline into smaller time periods (1–2 weeks)
- Each sprint has specific goals (e.g., complete X, Y, Z)
- Helps prioritize and organize tasks
- Incomplete tasks roll over to the next sprint

---

### Standup Meetings:
- Short, regular progress check-ins (not planning meetings)
- Purpose:
  - Accountability
  - Communication
  - Progress tracking
- Typical updates:
  - Progress made
  - Need more time
  - Offer help to others
- Important:
  - Be honest (e.g., midterm workload)
- If behind:
  - Adjust/restructure goals

#### Recommendations:
- Schedule before or after class
- Can be:
  - In-person
  - Virtual
  - Async (e.g., Slack channel)
- Slack bot exists but not very useful
- Suggested: create a dedicated **#standups channel**
