# Larpmaxxers Team Charter

## 1. Team Purpose And Objectives

Our team's purpose is to collaborate effectively to successfully complete CSE 110's course projects while also developing core software engineering skills like code reviews, testing, and agile development. Our objectives are:

- learn how to collaborate in a large team setting to be familiar with industry practices
- build a supportive team environment
- write clean and maintable code
- make a complete and polished final project that demonstrates our skills and understanding of software engineering principles

## 2. Common Values And Standards

- Communication:
  - Communicate clearly and respectfully
  - Teammates are expected to respond within 24 hours or let the group know if they are unavailab e
- Respect:
  - Respect everyone's opinions and ideas
  - Give constructive feedback and be open to receiving it
- Collaboration:
  - Be willing to help each other
  - Be open to different approaches and ideas

## 4. Potential Issues and Conflict-Resolution Strategies

- Uneven workload distribution
- lack of communication or miscommunication
- Missed deadlines
- Sudden personal emergencies/unforeseen circumstances that make a member unavailable for a period of time

Conflict resolution strategies:

- Discuss the concerns in team meetings
- Reassign tasks if needed
- Make sure members understand each other's strengths and weaknesses to better distribute work
- If a member is unavailable for a period of time, the team will work together to redistribute their tasks and ensure that the project stays on track

## 5. Rule Violations

- Give friendly warnings for first 2 times
- For next vioilation, notify the team lead and have discussion with the entire team under `#team-discussion`
- If the issue persists, notify the TA

## 6. Transparent Communication Protocols

- Communication through Slack
  - Create clear channel purposes: `#announcements`, `#team-discussion`, `#help`, etc.
- Reply in threads for:
  - Bug discussions
  - Code questions
  - Decisions
- Tag questions with urgency i.e. in `#help` channel
  - `[URGENT] Unable to return input to database.db`
- Use mentions for urgent issues and/or important announcements i.e. `@teammate`, `@here`, `@channel`
- If a member is unavailable for a period of time, they should update the team in the `#announcements` channel and set an out-of-office message on Slack
- For pull requests and code reviews, use the PULL_REQUEST_TEMPLATE.md in /.github/ to ensure that all necessary information is provided
